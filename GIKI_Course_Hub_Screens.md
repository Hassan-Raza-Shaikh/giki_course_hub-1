# GIKI Course Hub - Project Screens & Pages

This document provides a comprehensive list of all screens and interfaces within the GIKI Course Hub platform, including their primary functions.

## 1. Student-Facing Screens

### Landing Page
The initial entry point of the platform. It features high-fidelity visual design, live statistics (Course count, Material count, etc.), and a professional overview of the platform's utility.

### Google Sign-In Screen
Utilizes Firebase Google Authentication to provide a secure and seamless login experience. This screen is used to verify student identity before they can contribute materials or bookmark files.

### Explore Hub (Academic Directory)
A high-level view of the university structure. It allows users to navigate through different Faculties (e.g., FCSE, FGS) and their respective Programs (e.g., Computer Science, Data Science).

### Global Search Results
A specialized screen that appears when a user searches for a course. It provides instant filtering and redirection to specific course material pages.

### Course Material Page
The most important screen in the project. It includes:
- **Resource Grid**: Categorized tabs for Outline, Notes, Slides, Past Papers, and Lab Manuals.
- **File Preview & Download**: Allows users to interact with academic documents.
- **Reporting System**: A flagging interface to report incorrect or inappropriate content.

### Material Contribution Portal
An integrated upload interface where students can enter titles, select categories, and upload files directly to the platform's secure storage.

### User Dashboard / Bookmarks
A private area for the logged-in student to access all materials they have saved for quick reference.

---

## 2. Administrator Screens

### Admin Moderation Queue
The central hub for platform management. Admins use this screen to review pending file uploads from students and either approve them for public view or reject them.

### Content Report Dashboard
A specialized screen where admins review "Flags" sent by students. It allows them to quickly identify and remove problematic content.

### System Audit & Analytics
A summary screen showing platform growth, user activity logs, and administrative actions taken over time.
